## 为什么要使用 ref?

```js
为什么我们要使用带有 .value 的ref函数包裹器，而不直接使用普通的变量？为了解释这个问题，我们需要先了解Vue 3中的响应式系统是如何工作的。
当一个组件首次渲染的时候，Vue3会追踪 （track()）在渲染过程中使用的每一个ref，然后，当一个ref被修改的时候，它会触发 （trigger()） 追踪它的这个组件的一次重新渲染。
```

## reactive

```js
另外一种声明响应式状态的方式，ref()是将内部值包装在特殊对象中，这个特殊对象中通过get【依赖收集】和set【触发更新】去进行拦截操作，而reactive()则直接让对象本身就具备响应性。当ref()的值是一个对象时，会在内部调用reactive()进行包装。

```
