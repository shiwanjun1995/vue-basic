## 1、想要修改组件的默认样式，首先你需要知道组件是渲染在哪里的

### 是渲染在当前页面内部，还是脱离了 vue 的根节点单独渲染（根节点默认 id="app"）？

有些时候，我们有一些自定义的样式需求，修改 antdv 的样式，可以使用全局样式覆盖的方式。

但是这样会导致所有这些类名都会发生改变，所以我们需要在这一类的上层再加上一层类名，用于识别。

有些时候我们会碰到一种情况：我们想修改的 antdv 样式的上层它是挂载在 body 上（比如 Modal、Select 的下拉框），这种情况下，antdv 都会提供一个 API，以供使用者添加类名对其进行样式修改。

```js
组件弹框的 classname API 统一为 popupClassName。

AutoComplete 组件
Cascader 组件
Select 组件
TreeSelect 组件
TimePicker 组件
DatePicker 组件
Mentions 组件
<template>
  <a-select popupClassName="my-select-popup" />
</template>
```

## 2、类名scoped的限制

开发组件的时候，我们会设置一个该组件的scoped属性，这个表示限制css只能被当前文件的vue组件使用，那么它是如何实现这个限制的呢？

我们可以任意打开一个scoped的页面，打开调试，可以看到他们都有了一个共同的“标识”，这个是vue用于识别当前组件的唯一标识

```js
.parentClass >>> .sonClass
.parentClass /deep/ .sonClass
```

## 3、使用deep设置时，parentClass所在的节点是不带标识的怎么办？

antd vue的组件，渲染在#app外部，已经按照第一条给了自定义的类名wrapClassName，但是这个wrapClassName我们仍然无法使用，因为它所在的div并没有这个“标识”（极少数情况会出现）

这时我们拿出了终极武器，通过设置组件的 getContainer属性（有的组件是 getPopupContainer ），我们将原本渲染在#app外部的节点，手动挂载到了#app内部，之后再通过第二条的规则去设置就可以啦~

也可以不指定ref，直接设置 :getContainer="triggerNode => triggerNode.parentNode" 默认找到#app内部的父节点

总结：
以上三点，
第一点解决了没有父级类的问题（单独通过属性设置）
第二点解决了scoped造成的没有对应到父级类的问题（根据节点“标识”情况使用 >>> 或 /deep/）
第三点解决了没有父级类可以设置的问题（手动更改渲染规则）
